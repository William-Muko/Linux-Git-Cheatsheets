# Contributing Guidelines

## How to Contribute

1. **Fork** the repository
2. **Create** a feature branch (`git checkout -b feature/improvement`)
3. **Make** your changes
4. **Test** commands for accuracy
5. **Commit** with clear messages (`git commit -m 'Add new Linux command'`)
6. **Push** to your branch (`git push origin feature/improvement`)
7. **Submit** a pull request

## Guidelines

- **Keep it simple** - Focus on commonly used commands
- **Add examples** - Include practical usage examples
- **Test commands** - Verify all commands work as described
- **Follow format** - Match existing structure and style
- **Be clear** - Use descriptive commit messages

Open an issue for questions or suggestions before making large changes.
